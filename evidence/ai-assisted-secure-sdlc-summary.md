# AI-Assisted Secure SDLC Summary

## Purpose

This project demonstrates AI-assisted secure software delivery.

It supports security review, finding triage, release readiness checks, and evidence generation.

## Current Capabilities

- AI-assisted secure code review
- Dockerfile security review
- Kubernetes manifest review
- finding triage
- release security review
- markdown evidence generation
- GitHub Actions automation

## Safety Model

The current version uses local rule-based AI-style logic.

No real AI API key is required.

This keeps the project safe, repeatable, and suitable for CI/CD.

## Interview Explanation

I created this project to demonstrate how AI-assisted engineering can support Secure SDLC and release management.

It reviews code and configuration, identifies risky patterns, triages findings, and generates release readiness evidence.

It is intentionally advisory and keeps human approval as a required step for high-risk security and release decisions.
